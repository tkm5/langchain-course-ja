## 概要

このレクチャーでは，急速に進化するAI開発環境におけるFOMO（Fear of Missing Out: 取り残される恐怖）について議論します．Andrej Karpathyの投稿を引用しながら，この新しい時代にどう向き合うべきかを考えます．

## Andrej Karpathyの洞察

Karpathy氏は2025年末に以下のように述べました．

> プログラマーとして，これほど遅れを感じたことはない

この言葉は多くのエンジニアの共感を呼びました．エージェント，サブエージェント，プロンプト，コンテキスト，メモリ，MCP，LSP，スラッシュコマンド，ワークフロー，IDE統合など，新しいプログラミングレイヤーが次々と登場しています．

## ソフトウェアスタックの変化

ソフトウェアエンジニアリングは根本的に変化しています．

- コードを書く代わりに，コードを書くプロンプトを書く
- エンジニアはエージェントチームのオーケストレーターになりつつある
- チームリーダーがエンジニアにタスクを割り振るように，エージェントにタスクを割り振る
- 作業の重心が「実装」から「レビュー」に移行

## 抽象化の進化

ソフトウェアの抽象化は歴史を通じて進化してきました．

パンチカード → アセンブリ → C++ → Python → 自然言語

英語（自然言語）が新しい抽象レイヤーとなりつつあります．

## FOMOへの対処法

1. 好奇心を持つ: 好奇心が変化を推進し，限界を押し広げる原動力
2. 実験する: 手を動かし，実際に試してみることが最も重要
3. ノイズをフィルタリングする: すべてを追いかけず，関心のある問題に深く潜る
4. 進化の歴史を理解する: ReActプロンプト → Function Calling → エージェントワークフロー → Deep Agentsという進化の流れを把握する
5. FOMOを受け入れる: Karpathy氏でさえFOMOを感じている．誰もが同じ

## まとめ

- AI FOMOは業界全体が共有する感情であり，あなただけではない
- ソフトウェアスタックは根本的に変化し，エンジニアの役割はオーケストレーターに
- 好奇心と実験が最も重要なスキル
- すべてを追いかけるのではなく，関心のある領域に深く潜ることが効果的
- 進化の歴史を理解することでFOMOを軽減できる
