## 概要

LangChainとLlamaIndexは，LLMアプリケーションを構築するための2つの主要なフレームワークです．このレクチャーでは，両者の違いと類似点を比較し，プロジェクトに応じてどちらを選ぶべきかを解説します．

## 共通点

両フレームワークには多くの共通点があります:

- LLMアプリケーションを構築するためのユーティリティとツールを提供
- 外部データソースとの連携をサポート
- RAG（Retrieval Augmented Generation）の実装が可能
- エージェント機能をサポート

## 主な違い

### LangChain

- より高い人気と開発者への浸透率
- エージェント機能に対する包括的でロバストなサポート
- LangChain Expression Language（LCEL）による柔軟なRAGシステム構築
- 最新の研究に基づくエージェント機能の継続的なアップデート
- RAGとエージェントの両方をカバーする幅広い機能

### LlamaIndex

- データ取得と外部データ統合に特化
- よりデータ指向のアプローチ
- Reactアルゴリズムなどのエージェント実装をサポート
- エージェント機能は主に検索とデータ取得周りに集中

```text
┌─────────────────────────────────────────────────┐
│              LLMアプリケーション                  │
├────────────────────┬────────────────────────────┤
│   RAGアプリケーション  │   エージェントアプリケーション  │
│                    │                            │
│  LangChain: ◎      │  LangChain: ◎             │
│  LlamaIndex: ◎     │  LlamaIndex: △            │
│                    │                            │
│  データ連携         │  ツール選択・実行           │
│  質問応答           │  非決定的処理              │
└────────────────────┴────────────────────────────┘

◎ = 強力なサポート  △ = 基本的なサポート
```

## LLMアプリケーションの2つの種類

LLMアプリケーションは大きく2つに分類できます:

### 1. RAGアプリケーション

外部データと連携し，データに基づいた質問応答を行うアプリケーション．LangChainもLlamaIndexもこの領域をサポートしていますが，LangChainのLCELの導入により，RAGシステム構築の柔軟性が大幅に向上しました．

### 2. エージェントアプリケーション

LLMの推論能力を活用して，ツールの選択と実行を行う非決定的なアプリケーション．この領域ではLangChainがより包括的で，最新の研究にも積極的に対応しています．

## どちらを選ぶべきか

講師の推奨は以下の通りです:

- データ中心のアプリケーションであっても，LangChainを選択することを推奨
- エージェントアプリケーションの場合は，LangChainの方が遥かにロバストなエコシステムを持つ
- LangChainはRAGのニーズにも十分に対応できる

## まとめ

- LangChainとLlamaIndexは類似したフレームワークだが，それぞれ得意分野がある
- LlamaIndexはデータ取得・外部データ統合に特化
- LangChainはエージェント機能がより包括的で，最新の研究に対応
- RAGアプリケーションでもLangChainは十分な機能を提供
- 総合的にはLangChainの方が幅広いユースケースに対応可能
