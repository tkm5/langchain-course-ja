## 概要

このレクチャーでは，コースで使用するLangChainのバージョンと，バージョン互換性について説明します．セマンティックバージョニングの考え方と，バージョン差異への対応方法を解説します．

## LangChainのバージョン管理

コースはLangChain v1.0.2（収録時点の最新版）と互換性があります．`uv add langchain`コマンドでインストールすると，その時点の最新バージョンがインストールされます．

バージョンは`uv.lock`ファイルで確認できます．

## セマンティックバージョニング

```text
┌─────────────────────────────────────────┐
│   セマンティックバージョニング            │
│         major.minor.patch                │
│                                         │
│   例: 1 . 0 . 2                         │
│        │   │   └─ patch: バグ修正       │
│        │   └───── minor: 機能追加       │
│        └───────── major: 破壊的変更     │
└─────────────────────────────────────────┘
```

LangChainはセマンティックバージョニング（major.minor.patch）に従っています．

- パッチバージョンの変更（例: 1.0.2 → 1.0.7）: バグ修正や新機能追加．破壊的変更なし
- マイナーバージョンの変更: 後方互換性を保った機能追加
- メジャーバージョンの変更: 破壊的変更の可能性あり

多くの場合，マイナーバージョンの変更では互換性の問題は発生しません．破壊的変更がある場合は，講師がコースを更新して対応します．

## 問題が発生した場合

バージョンの互換性に問題がある場合は，Discordチャンネルで報告してください．講師が対応します．

## まとめ

- コースはLangChain v1.0.x と互換
- セマンティックバージョニングにより，パッチ更新では互換性が保たれる
- 破壊的変更がある場合はコースが更新される
- 問題があればDiscordで報告可能
